## 概述

DevMonitor 是一个 Windows 10/11 监控工具，通过 Telnet 连接远程设备，循环执行指定命令并将其回显展示在界面上。

### 核心约束

- 程序每次运行只连接一台设备，同时监控多台则启动多个程序实例
- 单文件 exe 发布，启动快、内存占用低

| 打包 | `go build` 单 exe，无外部依赖 |

## 功能规格

### 连接

- 协议：Telnet，端口由用户输入（无默认值）
- 认证：无，TCP 连接建立成功即视为已登录
- 编码：硬编码 UTF-8
- 连接中断：在显示区追加提示信息（如"连接已断开"），程序不自动重连，等待用户手动操作

### 监控命令

- 仅支持一条监控命令
- 用户点击"开始"按钮后：建立 Telnet 连接 → 进入监控循环
- 点击"开始"后，Telnet 连接建立成功即立即发送第一条命令（不等间隔）
- 循环逻辑：
  1. 发送监控命令
  2. 等待设备回显完整（以 `#` 字符结尾判定完整）
  3. 将回显内容按当前显示模式展示
  4. 等待用户配置的刷新间隔后，回到步骤 1
- 回显结束标志：硬编码为 `#`

### 刷新间隔

- 用户可配置
- 范围：0.01 秒 – 2 秒
- 默认值：0.5 秒
- 计时起点：设备返回完整回显之后

### 超时

- 硬编码 10 秒
- 超时行为：中断监控循环，断开 Telnet，在显示区展示超时错误提示

### 错误处理

- 网络层错误（超时、连接断开、读写失败等）发生时：中断循环，断开 Telnet，在显示区展示错误提示
- 错误信息同时写入本地错误日志文件（`error.log`）
- 设备返回的任何文本内容均视为正常回显，不记录日志

### 显示模式

- **覆盖模式**：每次新回显到达时，先彻底清空显示区，再写入新内容
- **追加模式**：持续追加回显直到用户手动点击"清空显示"按钮
- 显示区为只读，不可编辑
- 配色：白底黑字
- 字体：Consolas, 14pt，等宽

### 停止

- 用户点击"停止"按钮后：停止监控循环，断开 Telnet 连接，显示区内容保留不变

### 状态提示

- 无独立状态指示控件
- 通过按钮置灰提示状态：
  - 空闲：开始可用，停止置灰
  - 监控中：开始置灰，停止可用

### 退出

- 仅用户手动关闭窗口才退出

### 配置持久化

- 不做任何配置记忆，每次启动从零开始填写

## 架构要点

### goroutine 模型

```
main goroutine ──→ UI 事件循环（walk）
     │
     ├──→ 读取 goroutine：持续从 Telnet 连接读取字节流
     │         ↓ 检测到 # 后，通过 channel 发送完整回显块
     │
     └──→ 定时 goroutine：等待回显完成信号 → 等待间隔 → 发送下一条命令
```

- 读取 goroutine 和定时 goroutine 通过 channel 与主 goroutine 通信
- 主 goroutine 通过 `walk.InvokeSync` 将数据安全地推送到 UI 线程

### 模块划分

```
main.go           → 入口，组装各模块
├── config.go     → 配置数据结构，所有可调参数集中定义
├── telnet.go     → Telnet 连接、IAC 协商、收发字节流（纯协议层）
├── monitor.go    → 监控循环、超时控制、回显完整性判定（业务逻辑）
├── display.go    → 显示缓冲区管理，提供 Overwrite() / Append() / Clear()
├── logger.go     → 错误日志写入
└── ui.go         → 窗口布局、控件创建、事件绑定
```

| 文件 | 职责 | 对外暴露 |
|---|---|---|
| `config.go` | 默认间隔、超时时间、结束标志、字体字号、窗口尺寸等所有可调常量 | `Config` 结构体 |
| `telnet.go` | `Conn` 结构体，封装 `net.Conn`；建立连接、IAC 协商（回复 WONT/DONT）、`ReadBytes(delim byte)` 读取直到定界符、`Write(cmd string)` 发送命令、`Close()` 断开 | `NewConn(addr string) (*Conn, error)` |
| `monitor.go` | `Monitor` 结构体，组装 Telnet 连接 + 命令 + 间隔 + 超时；`Start()` 启动循环、`Stop()` 停止；内部管理读取 goroutine 和定时 goroutine 的生命周期 | `NewMonitor(conn, cmd, interval) *Monitor`；`Start(displayCallback)`、`Stop()` |
| `display.go` | 纯数据层，管理回显缓冲区和显示模式（覆盖/追加），不依赖任何 UI 框架 | `Display` 结构体；`SetMode()`、`Write(text)`、`Clear()`、`Text()` |
| `logger.go` | 追加写 `error.log`，带时间戳 | `LogError(msg string)` |
| `ui.go` | walk 控件创建、布局、事件绑定；通过回调函数连接 UI 事件与业务逻辑 | — |

### 编码原则

- **单一职责**：每个 `.go` 文件只做一件事，结构体方法只操作自己的字段
- **依赖方向**：`ui.go` → `monitor.go` → `telnet.go`；`display.go` 和 `logger.go` 为横向工具模块，被各层按需使用
- **UI 与逻辑分离**：`ui.go` 不包含任何 Telnet 协议或监控循环逻辑，只负责控件和事件转发；`display.go` 是纯数据层，不含任何 walk 控件的引用
- **channel 通信**：goroutine 之间只用 channel 传递数据，不用共享内存加锁
- **回调而非轮询**：监控循环通过回调函数将回显推送给 UI，而不是 UI 去拉取
- **常量集中**：所有可调参数（超时、间隔范围、字体、窗口尺寸等）集中在 `config.go` 中定义，不散落在各处
- **导出的都是接口意图**：每个包对外只暴露必要的类型和函数，内部实现细节用小写

### 错误日志

- 文件：程序同目录下的 `error.log`
- 格式：`[时间] 错误描述`
- 追加写入

## 注意事项

- Telnet IAC 协商：收到 IAC 后默认回复 WONT/DONT 拒绝所有选项协商，确保通信保持在纯文本模式
- 当用户快速连续点击"开始"时需做防重复点击处理
- 窗口关闭时需确保 Telnet 连接正确释放、goroutine 退出
- 数字输入（端口、间隔）需做合法性校验
