重要提示：

- 写任何代码前必须完整阅读design.md
- 每完成一个重大功能或里程碑后，必须更新 architecture.md
- 每完成一个开发步骤后，必须提交改动（git commit）并推送到 GitHub（git push）
- 每个开发步骤完成后，必须通过 `go build` 编译验证，编译不通过不可提交
- 开始任何编码工作前，先检查 git status 和项目文件结构，确认当前进度
- 代码标识符（函数名、类型名、变量名）使用英文；注释和文档使用中文
- commit message 格式：`<模块名>: <改动简述>`（如 `telnet.go: 实现 IAC 协商处理`）
- 不引入 design.md 技术选型表之外的第三方库，新增依赖需先确认
